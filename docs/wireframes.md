# MoneyWise App Wireframes

## Navigation Structure

### Main Tab Navigation
```
┌─────────────────────────────────────────────────────────────────┐
│                    Bottom Tab Navigation                        │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────┤
│    Home     │  Budgets    │Transactions │   Goals     │Settings │
│    🏠       │    📊       │    📝       │    🎯       │   ⚙️    |
└─────────────┴─────────────┴─────────────┴─────────────┴─────────┘
```

### Additional Screens (Accessible via Settings or Home)
- **Net Worth Tracking** - Accessible from Home dashboard or Settings
- **Account Management** - Accessible from Settings → Account Settings
- **Backup & Restore** - Accessible from Settings → Data Management
- **Receipt Scanning** - Accessible from Add Transaction or dedicated button
- **Reports & Analytics** - Accessible from Budgets or Settings
- **Receipt Library** - Accessible from Receipt Scanning or Settings
- **Data Export** - Accessible from Settings → Data Management
- **Family Budget Management** - Accessible from Settings → Family Settings
- **Investment Tracking** - Accessible from Net Worth or Settings
- **Advanced Analytics** - Accessible from Reports & Analytics

## Screen Flow Summary

### Main Navigation Flow
1. **Home Dashboard** - Main landing screen with overview and quick actions
2. **Budgets** - Budget management and tracking
3. **Transactions** - Transaction list and management
4. **Goals** - Savings goals and progress tracking
5. **Settings** - App configuration and additional features

### Feature Access Flow

#### Net Worth Tracking
- **Primary Access**: Home Dashboard → "Net Worth" quick action
- **Secondary Access**: Settings → Account Settings → Net Worth
- **Screens**: Net Worth Tracking, Add Asset/Liability

#### Account Management
- **Primary Access**: Settings → Account Settings → Bank Accounts
- **Screens**: Account Management, Add Account

#### Backup & Restore
- **Primary Access**: Settings → Data Management → Backup Data/Restore Data
- **Screens**: Backup & Restore, Data Export

#### Receipt Scanning
- **Primary Access**: Add Transaction → "Add receipt photo" option
- **Secondary Access**: Home Dashboard → "Scan Receipt" quick action
- **Screens**: Receipt Scanning, Receipt Library

#### Reports & Analytics
- **Primary Access**: Home Dashboard → "View Reports" quick action
- **Secondary Access**: Budgets → Reports tab
- **Screens**: Reports & Analytics, Data Export

### Screen Relationships

#### Core Features
- **Home Dashboard** → Quick access to all major features
- **Budgets** → Category management, budget insights
- **Transactions** → Add transaction, receipt scanning
- **Goals** → Goal tracking, progress visualization
- **Settings** → App configuration, data management

#### Advanced Features
- **Net Worth** → Asset/liability management, financial health overview
- **Account Management** → Multiple account support, transfers
- **Backup & Restore** → Data security, export/import
- **Receipt Scanning** → OCR processing, receipt library
- **Reports & Analytics** → Financial insights, data export

### Developer Implementation Notes

#### Navigation Implementation
- Use React Navigation with bottom tabs for main navigation
- Implement stack navigation for feature screens
- Use modal navigation for add/edit screens
- Implement deep linking for quick actions

#### Screen Dependencies
- **Net Worth** depends on Account Management for account data
- **Receipt Scanning** integrates with Transactions for automatic transaction creation
- **Backup & Restore** affects all data screens
- **Reports & Analytics** aggregates data from all other screens

#### Data Flow
- All screens share common data store (Redux/Context)
- Real-time updates across all screens
- Offline-first architecture with sync capabilities
- Secure data storage with encryption

#### UI/UX Considerations
- Consistent design language across all screens
- Responsive design for different screen sizes
- Accessibility features for all users
- Dark mode support throughout the app
- Loading states and error handling for all features
